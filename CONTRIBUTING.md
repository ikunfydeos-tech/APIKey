# 贡献指南

感谢你考虑为 API Key Manager 做贡献！

## 如何贡献

### 报告 Bug

如果你发现了 Bug，请通过 [GitHub Issues](../../issues) 提交报告，并使用 Bug 报告模板。

### 提出新功能

如果你有新功能的想法，欢迎通过 [GitHub Issues](../../issues) 提交，使用功能请求模板。

### 提交代码

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

## 开发指南

### 代码规范

- Python 代码遵循 PEP 8 规范
- 使用有意义的变量名和函数名
- 添加必要的注释

### 提交信息规范

使用清晰、有意义的提交信息：

- `feat: 添加新功能`
- `fix: 修复 Bug`
- `docs: 文档更新`
- `refactor: 代码重构`
- `style: 代码格式调整`
- `test: 测试相关`

### 测试

提交代码前，请确保：

1. 本地测试通过
2. 新功能有对应的测试用例
3. 没有引入新的警告或错误

## 代码审查

所有 PR 都需要经过代码审查才能合并。请耐心等待审查结果。

## 许可证

提交代码即表示你同意将代码以 MIT 许可证开源。
